기존 몬스터 시스템의 주요 문제점
  • 이동 경로의 단조로움: 대부분 Math.atan2(dy, dx) 직선 플레이어 추적 기반으로, 미로 벽 모서리에 잘 갇히고 단조로운 유인 플레이에 취약함.
  • 몬스터 간 협응(Cooperation) 부재: 각 몬스터가 서로 인식하지 않고 독립적으로 움직이며, 플레이어를 포위하거나 탱커 뒤에서 원거리 사격을 하는 전술이 없음.
  • 환경(미로) 활용 능력 결여: 미로라는 훌륭한 지형 요소가 있음에도 불구하고 벽을 엄폐물로 삼거나 기습하는 AI가 부재함.
  • 3,800줄 모놀리식 단일 switch 구조: Monster.prototype.update 내에 모든 몬스터 및 보스 로직이 뒤섞여 유지보수 및 신규 패턴 추가 난이도가 높음.

다양한 몬스터 개별 행동 AI 6대 원칙 설계
단순 추적을 넘어 전술적 깊이를 부여하는 몬스터 의사결정 알고리즘 원칙
1. 매복 & 은신 접근 (Ambush & Stealth)
Stalker / Assassin AI Pattern
전술형
플레이어의 시야(벽 뒤나 일정 거리 밖)에 있을 때는 은신(투명도 90% 감소) 상태를 유지하다가, 플레이어가 근접하거나 벽 코너를 도는 순간 초고속 기습 돌진을 시도합니다.
핵심 알고리즘 원칙
- Distance > 150px: Cloaking Mode (이동속도 0.5x, 반투명, 플레이어 조준선 회피)
- Distance ≤ 120px & 시선(Raycast) 노출 시: Ambush Strike (이동속도 3.0x, 잔상 유발, 무조건 타격)
2. 원거리 카이팅 (Tactical Kiting & Distance)
Ranger / Sniper AI Pattern
사거리 유지
플레이어에게 무작정 다가가지 않고, 최적 사거리(예: 250px)를 유지합니다. 플레이어가 접근하면 반대 방향으로 후퇴(Kiting)하며, 벽을 뒤에 두면 옆으로 돌아가 사격 각도를 확보합니다.
핵심 알고리즘 원칙
- Distance < MinRange: 플레이어 반대 벡터(dy, dx)로 후퇴 이동
- MinRange ≤ Distance ≤ MaxRange: 제자리 정지 후 정밀 조준 사격
- Wall Collision 시: Tangential Vector(측면 미끄러짐)로 수평 이동 전환
3. 벽/지형 활용 및 페이징 (Terrain & Wall Action)
Wall Walker / Phase Crawler
미로 특화
미로의 벽을 무시하고 벽 속을 통과하거나, 벽에 달라붙어 이동 속도가 2배가 되는 지형 특화 AI입니다. 플레이어가 예상하지 못한 벽너머에서 불쑥 등장해 미로 통로를 압박합니다.
핵심 알고리즘 원칙
- Wall Surface Contact Check: isPositionInWall() 검사
- Wall Jumper: 쿨다운마다 벽을 향해 도약하여 벽 반대편 미로 복도로 도약 재착지
- Phase Ghost: 3초 간격으로 1.5초간 벽 통과(Noclip) 상태 전환
4. 군집 반응 & 진형 형성 (Pack Rally & Formations)
Swarm / Pack Commander AI
협동 연계
동료 몬스터가 피격당하면 주변 몬스터들이 분노(Rage) 상태가 되어 공격속도가 증가하거나, 탱커 몬스터 뒤로 집결하여 플레이어 탄환을 막으며 진격하는 그룹 행동 시스템입니다.
핵심 알고리즘 원칙
- Flocking Steering Behavior (Boids 알고리즘: 응집, 분리, 정렬 적용)
- Leader-Follower Logic: 탱커/엘리트 몬스터 좌표를 보스/지휘관으로 삼아 호위 대형 유지
5. 구역 통제 & 장판 생성 (Zone Control & Hazards)
Trapper / Area Denial AI
지형 조작
직접 공격 대신 미로 바닥에 독성 포자 장판, 슬로우 젤, 전기 덫을 설치하여 플레이어의 무빙 동선을 제한하고 사지로 몰아넣는 전술입니다.
핵심 알고리즘 원칙
- Player Path Predict: 플레이어가 이동하는 진행 방향 길목에 덫 투척
- Area Persistence: 5~10초간 지속되는 틱 데미지/감속 필드 인스턴스 생성
6. 반응형 반사 & 방패 (Counter & Barrier)
Reflector / Vanguard AI
카운터형
전면에 에너지 방패를 펼쳐 플레이어의 탄환을 반사하거나 차단합니다. 방패가 켜진 동안 정면 타격은 무효화되며, 측면이나 후면을 공략해야만 피해를 줄 수 있습니다.
핵심 알고리즘 원칙
- Shield Angle Check: 탄환 충돌 각도와 몬스터 바라보는 각도의 차이 < 60deg 일 때 방사/차단
- Bullet Reflect: 플레이어 탄환의 vx, vy 벡터를 반대로 역전시켜 역공

코드 구조 및 리팩토링 모듈화 설계안
모놀리식 switch 문 구조에서 FSM (State Pattern) & Strategy Pattern 으로의 전환

🏗️ 모듈화 리팩토링 핵심 구상

기존 Monster 클래스의 update() 내부에 몰려있던 AI 로직을 독립된 AI Strategy 모듈 및 State Machine으로 분리합니다.

  • MovementStrategy: Chase, Kiting, Ambush, WallBounce, Flocking 전략 분리
  • AttackStrategy: SingleShot, ScatterShot, LaserCharge, TrapPlacement, ReflectShield 분리
  • FSM State Engine: IDLE, CHASE, ATTACK, RETREAT, STUN, SPECIAL_SKILL 상태 전이 구조 적용

📊 클래스 관계 구조

Monster (Class) ├── fsm: MonsterFSM │ ├── currentState: State (IdleState, ChaseState, KitingState...) │ └── transitionTo(newState) ├── moveStrategy: IMovementStrategy │ ├── StealthMoveStrategy │ ├── KitingMoveStrategy │ └── WallPhaseMoveStrategy └── attackStrategy: IAttackStrategy ├── ReflectAttackStrategy └── TrapDeployStrategy

💻 Proposed Code Pattern (예시 설계)

// 1. AI 행동 전략 인터페이스 패턴 예시 class KitingMoveStrategy { updatePosition(monster, player, mapEngine) { let dist = Math.hypot(player.x - monster.x, player.y - monster.y); let targetDist = monster.preferredRange || 250; if (dist < targetDist - 30) { // 카이팅: 플레이어 반대 방향 후퇴 let angle = Math.atan2(monster.y - player.y, monster.x - player.x); monster.x += Math.cos(angle) * monster.speed; monster.y += Math.sin(angle) * monster.speed; } else if (dist > targetDist + 30) { // 사거리 내 접근 let angle = Math.atan2(player.y - monster.y, player.x - monster.x); monster.x += Math.cos(angle) * monster.speed; monster.y += Math.sin(angle) * monster.speed; } } } // 2. Monster 클래스 등록 방식 monster.setStrategies({ move: new KitingMoveStrategy(), attack: new PlasmaSniperAttackStrategy() });
